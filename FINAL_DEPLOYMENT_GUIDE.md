# Ръководство за професионално изграждане и внедряване (Deployment Guide)

Това ръководство описва стъпките за превръщане на Вашия софтуер в професионален EXE пакет с лек инсталационен файл.

## 1. Подготовка на средата
Уверете се, че имате инсталирани необходимите библиотеки за пакетиране:
```bash
pip install pyinstaller winshell pypiwin32
```

## 2. Създаване на основния EXE пакет
Стартирайте автоматизирания скрипт за изграждане:
```bash
python build_app.py
```
- **Какво се случва:** Скриптът използва `ContractsApp.spec`, за да събере всички лога, шаблони (Word файлове) и базата данни за градовете в една папка.
- **Резултат:** В папка `dist/ContractsApp/` ще намерите готовото приложение.

## 3. Подготовка на "Леката" инсталация (Bootstrapper)
За да направите инсталацията лесна за потребителя:
1.  Компресирайте съдържанието на папка `dist/ContractsApp/` в ZIP файл с име **`ContractsApp.zip`**. (ZIP файлът трябва да е до инсталатора).
2.  Превърнете инсталационния скрипт в самостоятелно EXE (вече поддържа GUI и няма нужда от конзола):
    ```bash
    pyinstaller --onefile --noconsole --icon=vladpos_logo.png bootstrap_installer.py
    ```
- **Резултат:** В папка `dist/` ще получите `bootstrap_installer.exe`. Сега той използва прозорци за съобщения и избор на папка, което е много по-професионално.

## 4. Дистрибуция
За крайния потребител изпратете само двата файла:
- `bootstrap_installer.exe` (Инсталатор)
- `ContractsApp.zip` (Пакет с данни)

Когато потребителят стартира инсталатора, той автоматично:
1. Ще извлече файловете в `%LocalAppData%\ContractsApp`.
2. Ще създаде икона на работния плот (Desktop Shortcut).
3. Ще подготви работната среда (база данни и настройки).

---
**Забележка:** Вашият оригинален програмен код в папка `LD/` остава непроменен и напълно функционален за разработка. Новата логика за пътищата автоматично разпознава дали програмата работи от Python или като готово EXE.
